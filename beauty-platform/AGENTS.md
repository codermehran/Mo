# راهنمای Agent

این راهنما برای کل مخزن اعمال می‌شود.

- تمام مستندات و فایل‌های Markdown را به فارسی رسمی بنویسید.
- READMEها باید ساختارمند باشند و دست‌کم شامل بخش‌های «معرفی»، «پشته و معماری»، «ساختار مخزن»، «راه‌اندازی سریع با Docker»، «راه‌اندازی توسعه محلی»، «متغیرهای محیطی»، «تست و کیفیت»، و «جریان‌های کلیدی» باشند.
- فهرست متغیرهای محیطی را در جدول Markdown با ستون‌های «متغیر»، «مقدار نمونه/پیش‌فرض» و «توضیح» ارائه کنید.
- برای مراحل اجرایی از فهرست شماره‌دار استفاده کنید و فرمان‌ها را در بلوک کد سه‌تایی با برچسب زبان `bash` قرار دهید.
- توضیحات را کوتاه، شفاف و کاربردی نگه دارید و از جملات خبری استفاده کنید.
- پیام‌های کمیت گیت را امری و خلاصه بنویسید.
